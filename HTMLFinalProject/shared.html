Shared Hosting

Shared Hosting is when multiple websites share server space. The server owner, called the host, is responsible for the physical equipement and its maintainance, and the website owner is responsible for managing the site. This can be a great tool for buisnesses who can't afford the harware or just don't want to spend the energy and effort to host their own website. It is common for websites of all sizes to use Shared Hosting, but is more common for smaller sites.

Although the size of a hosts server can rance greatly, all resources are limited. The distrobution of those resources is usually outlined in a terms-of-service or contract with eash website owner. Different buisness have their own rules, but there are typically service limits on CPU usage, memory, bandwidth and server storage. If a website exceeds the limits set forth in their contract, services may be throttled or financial fees may apply. Using more than your allocated recources not only costs the host money, but it can negtively impact the other websites sharing those resources.

According to NameCheap.com, Shared Hosting is best for small buisness and blogs that dont requrie advanced settings or high bandwidth. Because the cost of maintaining a server are split up across many webiste owners, Shared Hosting can be cost effective and have the smallest barrier to enter. Some advantages to web hosting are:

  • Low start up cost: monthly fees can be much softer then large hardware purchases
  • No technical expertise requred, you are paying for the hardware and the maintainance
  • Frees you up to work in your specialty
  • Ease of use
  • Scalability
  • Uptime guarantees

Hosting.com points out that Shared Hosting is not a perfect solution for everyone, it can have some significant downsides:

  1. Less control and flexability: performance optimization is paaritally out of the website owners control
  2. Limited resources
  3. Security risks: its possible for other websites poor management to affect your site
